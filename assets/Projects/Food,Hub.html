Foodhub has been the longest and toughest of the project that I worked with. It involved around 450+ datasets that span over whole of the US.


Portal was built using Geoserver (acts as a data store), Openlayers (mapping library) and Postgresql (spatial database) along with numerous 3rd party libraries.


Project allowed user to view / manipulate multiple datasets, change it's visualizations on the go, filter the dataset using legend controls & so on. It offered user customizations from dots to icons to clusters and radius around the icons to help user understand the area it would serve. User could alter the radius, icon or even the color of the visualized layers. A unique feature called cross-walks was offered in the portal, where it allowed user to interactively select any administrative boundary (district, county, zip etc.) and cross walk it over another admin boundary to get an idea of the dynamic intersections data could offer.


Solution is a hub of data where majority of data was gathered from Census bureau, but manipulated and altered to fit needs for a socio economic decision making on any scale. Features like voronoi diagrams, cross walks were the talking points that portal offered.
